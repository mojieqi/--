### 6.4 AI 内容审核集成

**集成方案：** 帖子发布时自动触发审核流水线。

```
用户发布帖子 → CwPostController.create()
  → CwPostServiceImpl.publish(post)
    → 1. 保存帖子（状态=待审核）
    → 2. 异步调用 ContentAuditTool
         └── 关键词匹配 + LLM 深度审核
    → 3. 审核结果：
         ├── 通过(audit_status=1)：帖子正常展示
         └── 驳回(audit_status=2)：记录审核原因，帖子仅作者可见
    → 4. 返回审核结果给前端
```

| 审核维度 | 实现方式 | 来源 |
|---------|---------|------|
| 敏感词过滤 | 关键词库匹配（5类敏感词） | ContentAuditTool（已实现） |
| LLM 深度审核 | 调用 content_audit 提示词模板 | Phase 1 已就绪 |
| 违规识别 | 违法/人身攻击/色情/广告/校园安全 | 5类敏感词库 |

### 8.1 AI 智能推荐引擎

**推荐算法设计：**

```
推荐策略（多因子加权）：
├── 内容匹配（40%）：LLM 分析用户帖子语义，匹配相似内容
│   └── 基于 content_recommend 提示词模板（Phase 1 已就绪）
├── 热度权重（25%）：基于浏览数+点赞数+评论数的动态热度分
│   └── 热度分 = log(view_count+1)*1 + like_count*3 + comment_count*5
│   └── 时间衰减因子：1 / (1 + hours_ago / 24)
├── 多样性（20%）：确保推荐覆盖不同分类，避免信息茧房
│   └── 同分类最多展示3条
├── 时效性（10%）：24小时内发布的帖子提权
└── 个人化（5%）：根据用户历史浏览/互动偏好调整
```
